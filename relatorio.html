O que são frameworks e para que são utilizados?

Um framework é como um esqueleto ou uma estrutura de base pronta para o desenvolvimento de software. Ele reúne códigos genéricos, bibliotecas e diretrizes que evitam a criação de sistemas do zero. Eles são amplamente usados para acelerar, padronizar e dar segurança a projetos de programação.

Frameworks para diferentes linguagens

Java

Framework: Micronaut

Ambientes Cloud-Native

Desenvolvido do zero focado em microsserviços, serverless e Kubernetes

Baixo Consumo de Memória

Usa principalmente injeção de dependências no tempo de compilação

Inicialização Rápida

Ideal para aplicações que necessitam de auto-scaling

PHP

Framework: Laravel

MVC

Segue o padrão Model, View, Controller, herdado do Rails (framework de Ruby)

Eloquent ORM

ORM (object-relational mapping) inclusa no framework, agilizando tarefas comuns (como a criação de CRUDs)

Filosofia "Batteries-Included"

Vem com diversas features built-in (de fábrica), como injeção de dependência, routing, soluções para websockets, etc.

Link do site da framework

Python

Framework: Flask

Curva de Aprendizado Suave

Por ter menos complexidade, é um framework mais fácil de aprender do que algo mais complexo, como Django

Altamente Customizável

Por ser pouco opinado, permite escolher exatamente as ferramentas e extensões que serão utilizadas

Ideal para serviços RESTful simples

Muito utilizado para criar APIs RESTful que se comunicam com outras aplicações

Link do site da framework

Javascript / Typescript

Framework: Fastify

Focado em Velocidade

Focado em performance no servidor, se diferenciando de outros frameworks JS, como ExpressJS

Low-Overhead

Segue a filosofia low-overhead, que foca em remover camadas de abstração à fim de focar na velocidade e uso eficiente de recursos

TS First

Se vende como alternativa ao ExpressJS, principalmente focando em devs de TypeScript, que vem com suporte nativo, sem precisar de extensões ou plugins
